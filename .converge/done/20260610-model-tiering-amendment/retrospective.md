## 7. 经验教训

1. **轻量计划范式裁定（三方一致 + 设计审查补充）**：范式成立，但需三条可检验适用规则——(i) 声明低成本执行必须附逐字插入块；(ii) 治理对象必须附入边盘点；(iii)（设计审查新增候选）入边盘点必须含**出边**——新条款向既有 schema/义务体系输出了什么。范式规则需要 single source（候选项，走治理）。
2. **"计划违反自身原则"是自指类修宪的高发缺陷**：以合同确定性为论旨的计划自己没过确定性检查（U6）。修复方式（逐字插入块）使 R2 验收变成纯机械核对——验收确定性与执行确定性同源。
3. **时序倒置的处置先例**：用户自行前置执行+提交时，converge 的正确响应是事后字节级验收 + 如实入账，而非回滚或静默——产物正确性与流程合规性分开裁决。
4. **C1 分组效果观察（第 1/3-5 次记录）**：本次收敛 Orchestrator 按四分组定位职责——每轮必做组逐轮执行（#1/#2-4/#8/#9 全执行，无 overturn 事件）、条件触发组正确跳过未触发项（#7 预算未触上限、#13/#14 无 contract/外部输入）、Round 0 前置组整组跳过并记 skip_reason、收口组 #18 执行（强制）。主观体验：分组对"收口时该做什么"的定位帮助明显，对每轮循环帮助一般（条目本来就熟）。**自报告偏差警示**：此观察由被观察者本人记录，证据力弱，待跨收敛积累。
